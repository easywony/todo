# PR Checklist
> PR 생성 전 최종 검증 항목

---

## ■ REQUIRED

- [ ] base 브랜치가 `develop` 인가?
- [ ] 커밋 메시지에 `feat:` `fix:` `refactor:` `chore:` `docs:` prefix 있는가?
- [ ] PR 본문 템플릿이 작성되었는가?
- [ ] 하드코딩(색상/폰트/간격)이 없는가?
- [ ] `main` 브랜치에 직접 push하지 않았는가?

## □ OPTIONAL

- [ ] UI 변경이 있으면 스크린샷 또는 영상 첨부
- [ ] 관련 이슈 번호 연결 (`Closes #이슈번호`)
